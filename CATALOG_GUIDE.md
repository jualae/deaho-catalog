# CERAD CATALOG 이미지 관리 가이드

> 이 문서는 카탈로그 앱에서 사용하는 이미지와 데이터가 어디에 저장되어 있고, 이미지를 수정하거나 새로운 카테고리/이미지를 추가하려면 어떻게 해야 하는지를 초보자도 따라할 수 있도록 상세하게 설명합니다.

---

## 1. 전체 구조 이해하기

카탈로그 앱은 크게 **두 가지 데이터**로 구성됩니다.

| 구분 | 역할 | 저장 위치 |
|------|------|-----------|
| **catalog.json** | 카테고리 목록, 페이지 번호, 이미지 주소 등 모든 설정 정보 | 클라우드 (CloudFront CDN) |
| **이미지 파일들** (page_001.jpg ~ page_054.jpg) | 실제 카탈로그 페이지 이미지 | 클라우드 (CloudFront CDN) |

앱이 실행되면 먼저 `catalog.json` 파일을 인터넷에서 다운로드하고, 그 안에 적힌 이미지 주소(URL)를 보고 각 페이지의 이미지를 불러옵니다. 따라서 **이미지를 바꾸거나 추가하려면 클라우드에 올라가 있는 파일들을 수정**해야 합니다.

---

## 2. catalog.json 파일의 구조

`catalog.json`은 앱의 "설계도"와 같은 파일입니다. 현재 이 파일은 아래 주소에 저장되어 있습니다.

```
https://d2xsxph8kpxj0f.cloudfront.net/310519663337257885/g9FMjJFMH7rkmYiPvT29tr/cerad-catalog/catalog.json
```

이 파일을 웹 브라우저에서 열면 아래와 같은 구조를 볼 수 있습니다.

```json
{
  "version": "1.0",
  "title": "CERAD CATALOG",
  "subtitle": "대호상사 · Hair & Murin 전문 미용용품 종합 카탈로그",
  "totalPages": 54,
  "imageBaseUrl": "s3",
  "imageIds": {
    "1": "1XjV_nJgcC1Lh5jPW581AzlcmRTXvU7gn",
    "2": "10jqT2h9pMMtlLX1Zjqtoyaz3RvZiWITC",
    ...
  },
  "imageUrls": {
    "1": "https://d2xsxph8kpxj0f.cloudfront.net/.../images/page_001.jpg",
    "2": "https://d2xsxph8kpxj0f.cloudfront.net/.../images/page_002.jpg",
    ...
  },
  "categories": [
    {
      "id": "iron",
      "num": 1,
      "name": "아이롱 / 매직기 / 고데기",
      "name_en": "IRON / MAGIC / CURLER",
      "icon": "iron",
      "color": "#C0392B",
      "pages": [3, 4, 5, 6, 7, 8],
      "desc": "전문 미용 아이롱, 매직기, 고데기 제품 라인업"
    },
    ...
  ]
}
```

각 항목의 의미는 다음과 같습니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| `version` | 카탈로그 버전 | `"1.0"` |
| `title` | 앱 상단에 표시되는 제목 | `"CERAD CATALOG"` |
| `subtitle` | 제목 아래 부제 | `"대호상사 · Hair & Murin..."` |
| `totalPages` | 전체 페이지 수 | `54` |
| `imageUrls` | 각 페이지 번호에 대응하는 이미지 URL | `"1": "https://...page_001.jpg"` |
| `imageIds` | Google Drive 백업 ID (보조용) | `"1": "1XjV_nJg..."` |
| `categories` | 카테고리 목록 (아래에서 자세히 설명) | 배열 형태 |

## 4. 이미지 파일의 저장 위치

모든 이미지는 아래 형식의 URL로 클라우드에 저장되어 있습니다.

```
https://d2xsxph8kpxj0f.cloudfront.net/310519663337257885/g9FMjJFMH7rkmYiPvT29tr/cerad-catalog/images/page_XXX.jpg
```

여기서 `XXX`는 3자리 페이지 번호입니다. 예를 들어 페이지 1번 이미지는 `page_001.jpg`, 페이지 54번 이미지는 `page_054.jpg`입니다. 이 파일들은 **AWS CloudFront CDN**이라는 클라우드 서비스에 저장되어 있으며, 직접 접근하려면 특별한 권한이 필요합니다. 하지만 걱정하지 마세요. 저(Manus)에게 요청하시면 이미지를 업로드하고 catalog.json을 업데이트하는 작업을 대신 처리해 드릴 수 있습니다.

## 5. 기존 이미지를 교체(수정)하는 방법

기존 카탈로그 페이지의 이미지를 새 이미지로 바꾸고 싶다면, 다음과 같이 요청해 주세요.

### 요청 방법

**저에게 채팅으로 아래 내용을 알려주세요.**

1. **바꿀 페이지 번호** — 예: "페이지 5번 이미지를 교체하고 싶어요"
2. **새 이미지 파일** — 채팅에 이미지를 첨부해 주세요 (JPG 또는 PNG)

### 처리 과정 (제가 대신 해드립니다)

제가 요청을 받으면 아래 과정을 자동으로 처리합니다.

1. 새 이미지를 클라우드(S3)에 업로드합니다.
2. `catalog.json`에서 해당 페이지의 이미지 URL을 새 URL로 변경합니다.
3. 업데이트된 `catalog.json`을 클라우드에 다시 업로드합니다.
4. 앱을 다시 열면 새 이미지가 반영됩니다.

### 예시 요청

> "페이지 10번 이미지를 이 사진으로 바꿔주세요" (이미지 첨부)

## 6. 기존 카테고리에 새 이미지를 추가하는 방법

예를 들어 "브러시" 카테고리에 새로운 제품 페이지를 추가하고 싶다면 다음과 같이 요청해 주세요.

### 요청 방법

**저에게 채팅으로 아래 내용을 알려주세요.**

1. **어떤 카테고리에 추가할지** — 예: "브러시 카테고리에 추가하고 싶어요"
2. **새 이미지 파일** — 채팅에 이미지를 첨부해 주세요 (여러 장도 가능)
3. **어디에 넣을지** — 예: "브러시 카테고리 맨 뒤에 추가" 또는 "페이지 12번과 13번 사이에 추가"

### 처리 과정 (제가 대신 해드립니다)

1. 새 이미지를 클라우드에 업로드합니다.
2. `catalog.json`의 `imageUrls`에 새 페이지 번호와 URL을 추가합니다.
3. 해당 카테고리의 `pages` 배열에 새 페이지 번호를 추가합니다.
4. `totalPages` 숫자를 업데이트합니다.
5. 업데이트된 `catalog.json`을 클라우드에 다시 업로드합니다.

### 예시 요청

> "브러시 카테고리 맨 뒤에 이 이미지 2장을 추가해주세요" (이미지 2장 첨부)

## 7. 완전히 새로운 카테고리를 추가하는 방법

새로운 제품 카테고리를 만들고 싶다면 다음 정보를 알려주세요.

### 요청 시 필요한 정보

| 항목 | 설명 | 예시 |
|------|------|------|
| **카테고리 한글 이름** | 앱에 표시될 이름 | "드라이기" |
| **카테고리 영문 이름** | 영문 표기 | "HAIR DRYER" |
| **카테고리 설명** | 간단한 설명 문구 | "전문 미용 드라이기 제품 라인업" |
| **원하는 색상** | 카테고리 테마 색상 (없으면 제가 선택) | "파란색" 또는 "#3498DB" |
| **이미지 파일들** | 해당 카테고리에 들어갈 이미지들 | 이미지 첨부 |

### 예시 요청

> "새 카테고리를 만들어주세요. 이름은 '드라이기', 영문은 'HAIR DRYER', 설명은 '전문 미용 드라이기 제품 라인업'이고 색상은 파란색으로 해주세요. 이미지는 이 3장입니다." (이미지 3장 첨부)

## 9. 자주 묻는 질문 (FAQ)

**Q: 이미지 파일 크기나 형식에 제한이 있나요?**

JPG 또는 PNG 형식을 권장하며, 파일 크기는 가능하면 **5MB 이하**가 좋습니다. 너무 큰 이미지는 앱에서 로딩이 느려질 수 있습니다. 가로 세로 비율은 기존 카탈로그 이미지와 비슷하게 맞추면 가장 보기 좋습니다.

**Q: 이미지를 바꾸면 앱을 다시 설치해야 하나요?**

아닙니다. 앱을 완전히 종료했다가 다시 열면 자동으로 최신 데이터를 불러옵니다. 앱을 다시 설치할 필요는 없습니다.

**Q: 카테고리 순서를 바꿀 수 있나요?**

네, 가능합니다. "카테고리 순서를 바꿔주세요. 브러시를 1번으로, 아이롱을 2번으로..." 와 같이 요청해 주시면 됩니다.

**Q: 직접 catalog.json을 수정할 수 있나요?**

기술적으로는 가능하지만, JSON 형식을 정확히 지켜야 하고 클라우드에 업로드하는 과정이 필요하므로 **저에게 요청하시는 것을 권장**합니다. 실수로 형식이 깨지면 앱이 작동하지 않을 수 있습니다.

**Q: 한 번에 여러 이미지를 교체하거나 추가할 수 있나요?**

네, 한 번에 여러 장을 첨부하고 각각 어디에 들어갈지 알려주시면 한꺼번에 처리해 드립니다.

## 10. 요약: 무엇이든 저에게 요청하세요

이미지나 카테고리와 관련된 모든 변경 작업은 **채팅으로 요청**하시면 됩니다. 복잡한 기술적 작업은 제가 대신 처리해 드리니, 아래 내용만 알려주세요.

| 하고 싶은 것 | 알려주실 내용 |
|-------------|--------------|
| 이미지 교체 | 페이지 번호 + 새 이미지 첨부 |
| 이미지 추가 | 카테고리 이름 + 위치 + 새 이미지 첨부 |
| 새 카테고리 생성 | 이름 + 영문명 + 설명 + 색상 + 이미지 첨부 |
| 카테고리 정보 수정 | 카테고리 이름 + 바꿀 내용 |
| 카테고리 순서 변경 | 원하는 순서 |
| 카테고리 삭제 | 삭제할 카테고리 이름 |
